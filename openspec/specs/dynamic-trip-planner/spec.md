## ADDED Requirements

### Requirement: 多维度行程规划输入
系统 SHALL 接受以下输入参数进行行程规划：出发地、目的地（或历史路线）、旅行天数、预算范围、同行人数、偏好标签（文化/美食/自然/购物）、出行方式。

#### Scenario: 规划长安三日游
- **WHEN** 用户提交：目的地=西安、天数=3、预算=3000元/人、偏好=文化+美食
- **THEN** 生成包含景点、餐饮、住宿的完整三日行程

### Requirement: 预算动态分配
系统 SHALL 根据用户总预算智能分配：住宿占比、餐饮占比、门票占比、交通占比、购物占比。SHALL 在每个类别内推荐符合预算的具体选项。

#### Scenario: 低预算行程推荐
- **WHEN** 用户预算为 1500 元/人/3天
- **THEN** 推荐经济型酒店、特色小吃而非高档餐厅、免费/低价景点优先

### Requirement: 酒店推荐与预订
系统 SHALL 根据行程路线推荐沿途酒店，考虑因素：位置（距景点距离）、价格、评分、历史主题（优先推荐有历史特色的酒店/民宿）。

#### Scenario: 推荐西安历史主题酒店
- **WHEN** 行程包含汉长安城遗址
- **THEN** 优先推荐城墙附近或仿古风格的酒店/民宿

#### Scenario: 跳转到预订
- **WHEN** 用户选择推荐酒店并点击预订
- **THEN** 跳转到 OTA 平台（携程/美团）完成预订

### Requirement: 门票推荐与购买
系统 SHALL 推荐行程中涉及的历史景点门票，包含：景点名称、票价、开放时间、游览建议时长。

#### Scenario: 推荐门票
- **WHEN** 行程包含"汉长安城遗址"和"陕西历史博物馆"
- **THEN** 推荐门票：陕西历史博物馆(免费需预约)、汉城湖景区(30元)

### Requirement: 实时行程调整
系统 SHALL 支持用户在旅行过程中动态调整行程（如增加/删除景点、修改预算），后端 Agent SHALL 实时重新规划剩余行程。

#### Scenario: 临时增加景点
- **WHEN** 用户在行程第 2 天请求增加"大雁塔"
- **THEN** Agent 重新规划第 2-3 天行程，调整时间分配和后续酒店

#### Scenario: 天气导致行程变更
- **WHEN** 行程当天遇大雨（通过天气 API 检测）
- **THEN** Agent 主动建议将户外景点替换为室内博物馆

### Requirement: 行程规划 Agent 对话交互
系统 SHALL 提供自然语言对话界面，用户可通过对话方式调整行程（如"我想多住一天""能不能换成更便宜的酒店"）。

#### Scenario: 对话调整预算
- **WHEN** 用户说"预算减到 2000，去掉购物安排"
- **THEN** Agent 重新规划行程，降低住宿和餐饮标准，移除购物站点
